# 电机控制专家智能体 PRD（V0.1）

## 1. 文档目标

本文档定义一套面向电机控制器研发场景的专家智能体产品需求，覆盖以下任务：

1. 设计方案建议
2. 原理图审核
3. Layout 审核
4. 风险分析
5. 测试用例输出
6. 测试报告审核

目标不是一次性做成“全能专家”，而是先形成一个可迭代、可评测、可追溯的工程智能体平台，并能够随着大模型、知识库和规则库演进持续升级。

---

## 2. 产品定位

### 2.1 产品定义

该产品是一套“专家工作流系统”，而不是单一聊天机器人。系统通过以下能力协同工作：

1. 企业知识检索
2. 高质量外部知识检索
3. 工程规则校验
4. 多阶段推理与证据归因
5. 结构化输出与人工复核
6. 评测闭环与版本回归

### 2.2 目标用户

1. 电机控制器硬件工程师
2. 电机控制算法工程师
3. PCB Layout 工程师
4. 测试工程师
5. 技术经理与评审专家
6. 新人培养与知识传承团队

### 2.3 目标对象

被驱动电机类型包括：

1. 步进电机
2. 无刷直流电机（BLDC / PMSM）
3. 伺服电机
4. 直线电机

---

## 3. 业务目标

### 3.1 核心目标

1. 降低电机控制器方案设计和评审对少数专家的依赖
2. 提高原理图、Layout、测试方案与测试报告的评审一致性
3. 将内部历史经验、飞书知识库和外部高质量资料转化为可复用能力
4. 建立可量化评测体系，使模型升级和规则更新可控

### 3.2 成功标准

第一阶段建议以以下指标验收：

1. 重点问题召回率大于等于 80%
2. 高严重度问题漏报率小于等于 10%
3. 证据引用准确率大于等于 90%
4. 专家复核后可直接采用或轻微修改的建议比例大于等于 60%
5. 单次审核平均耗时相比人工下降 30% 以上

## 4. 范围定义

### 4.1 第一阶段范围（建议 8 到 12 周）

优先支持：

1. 电机类型
   - BLDC / PMSM
   - 步进电机
2. 任务类型
   - 设计方案建议
   - 原理图审核
   - 风险分析
   - 测试用例输出

原因：

1. 输入资料最容易结构化
2. 内部经验最容易沉淀为规则
3. 最容易建立评测集

### 4.2 第二阶段范围

扩展支持：

1. Layout 审核
2. 测试报告审核
3. 伺服电机
4. 直线电机

### 4.3 第三阶段范围

1. 跨项目经验推荐
2. 设计空间探索与方案比选
3. 自动生成评审记录与问题单
4. 与 PLM、ALM、缺陷管理系统联动

## 5. 用户故事

### 5.1 设计方案建议

作为硬件工程师，我希望输入电机类型、功率、电压、控制目标、传感器方案、成本约束、环境条件后，系统能给出拓扑建议、关键器件建议、保护建议和风险提示，并引用企业内部案例与外部资料。

### 5.2 原理图审核

作为评审专家，我希望上传原理图 PDF、网表、BOM、关键器件 datasheet 后，系统能按电源、驱动、采样、保护、隔离、接口、EMC、热设计等维度识别问题并给出严重度。

### 5.3 Layout 审核

作为 Layout 工程师，我希望系统基于 PCB 导出资料识别大电流回路、栅极回路、采样回路、隔离边界、散热路径、敏感信号走线、回流路径等潜在风险。

### 5.4 风险分析

作为技术经理，我希望系统输出设计风险清单、触发条件、后果、检测方法、缓解措施和剩余风险等级，支持项目早期评审。

### 5.5 测试用例输出

作为测试工程师，我希望系统根据产品规格、控制模式和风险点自动生成覆盖功能、性能、鲁棒性、保护和边界工况的测试用例。

### 5.6 测试报告审核

作为评审人，我希望系统读取测试报告后判断样本量、测试条件、判定准则、结果解释和缺陷闭环是否充分。

## 6. 输入与输出

### 6.1 典型输入

1. 项目规格书
2. 电机参数表
3. 系统框图
4. 原理图 PDF / 原始导出文件
5. 网表与 BOM
6. PCB 导出文件、层叠、Gerber、关键测量截图
7. 测试计划与测试报告
8. 飞书知识库文档
9. 企业历史项目复盘资料
10. 外部资料
    - 学术论文
    - TI / Microchip / ST / onsemi / Infineon 等厂商文档
    - 器件 datasheet
    - Reference design
    - 应用笔记

### 6.2 标准输出

每类任务输出都应统一包含：

1. 任务结论
2. 关键发现
3. 风险分级
4. 证据引用
5. 建议动作
6. 假设与缺失信息
7. 置信度
8. 结构化 JSON

---

## 7. 核心能力要求

### 7.1 知识能力

1. 读取并索引飞书知识库
2. 管理内部项目文档版本
3. 接入外部可信知识源
4. 区分知识来源级别与适用边界

### 7.2 工程推理能力

1. 基于电机类型识别关键设计关注点
2. 基于功率级、驱动方式、控制模式做差异化建议
3. 结合规则、文档和历史案例生成结论
4. 对建议给出约束条件和适用范围

### 7.3 审核能力

1. 对原理图分域审核
2. 对 Layout 分区域审核
3. 对风险项做 FMEA 风格分析
4. 对测试项做覆盖性和有效性分析

### 7.4 平台能力

1. 任务编排
2. 证据追溯
3. 权限控制
4. 评测与回归
5. Prompt 与规则版本管理
6. 模型路由与升级切换

---

## 8. 非功能需求

### 8.1 可追溯性

所有关键结论必须关联来源证据，至少记录：

1. 来源文档
2. 文档版本
3. 引用片段或定位信息
4. 推理所用规则版本
5. 模型版本

### 8.2 可控性

1. 高风险结论必须允许人工确认
2. 所有自动输出保留假设声明
3. 对缺失资料应主动提示，而不是臆断

### 8.3 可扩展性

1. 能新增电机类型
2. 能新增任务类型
3. 能替换底层模型
4. 能扩展外部知识源和企业系统连接器

### 8.4 安全与权限

1. 飞书文档遵守原有权限控制
2. 项目资料分级存储
3. 外部检索内容与内部资料严格区分
4. 不允许未经审计的写操作回写业务系统

## 11. 风险与应对

### 11.1 主要风险

1. 企业知识质量不一，导致错误迁移
2. 图纸资料结构化程度不够，影响审核效果
3. LLM 输出可信度不稳定
4. 规则与模型结论冲突时缺少裁决机制
5. 模型升级导致历史 prompt 行为漂移

### 11.2 应对策略

1. 建立知识分级和来源权重
2. 优先做可结构化任务
3. 以规则兜底关键问题
4. 引入专家复核工作流
5. 版本升级前强制跑评测集

---

## 12. 验收建议

建议采用“双轨验收”：

1. 产品验收
   - 可用性
   - 输出完整性
   - 响应速度
   - 证据可追溯
2. 专业验收
   - 问题召回率
   - 误报率
   - 建议质量
   - 严重度判断合理性

---

## 13. 本阶段建议结论

建议按以下策略启动实施：

1. 先做 BLDC / 步进电机
2. 先做方案建议和原理图审核
3. 先建立知识、规则、评测三件套
4. 再扩展 Layout、测试和更多电机类型

这是最稳妥、最容易获得正反馈、也最容易形成长期资产的路径。
